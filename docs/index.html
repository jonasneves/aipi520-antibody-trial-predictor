Project Overview

Real-world ML challenge: predicting clinical trial success for antibody therapeutics

Problem Statement i Why Antibody Trials?
Antibodies represent ~30% of the pharmaceutical development pipeline with high commercial value ($100B+ market), but face historically high failure rates. Their complexity and specificity make them ideal candidates for ML prediction, as subtle trial design factors significantly impact success.

Predict probability of success for Phase 2 & 3 antibody clinical trials. With 70% failure rate in Phase 2[1] and costs of $10-100M+[2], early prediction can save pharmaceutical companies significant resources.

Data Source

ClinicalTrials.gov API v2[3]

Completed Phase 2/3 antibody trials using query:

AREA[OverallStatus]COMPLETED AND (AREA[Phase]PHASE2 OR AREA[Phase]PHASE3)

Success Labeling Strategy i How Labels Are Assigned:
Automated labeling based on ClinicalTrials.gov OverallStatus field. Ambiguous statuses (RECRUITING, ACTIVE_NOT_RECRUITING, UNKNOWN) excluded from training to ensure clean labels. This conservative approach prioritizes label quality over dataset size.

Binary Classification Based on Trial Status:

  • Success (1): COMPLETED, APPROVED_FOR_MARKETING, AVAILABLE
  • Failure (0): TERMINATED, WITHDRAWN, SUSPENDED, NO_LONGER_AVAILABLE
  • Excluded: Ambiguous statuses (RECRUITING, ACTIVE_NOT_RECRUITING, etc.)

Evaluation Methodology

  • Train/Test Split: 80/20 with stratification i Why 80/20?
    Standard practice balancing model learning capacity with reliable test evaluation. 80% training data helps models learn complex patterns while 20% provides adequate test samples for performance estimation.[7]
  • Cross-Validation: 3-fold stratified CV on training set i Why Stratified?
    Preserves class distribution across folds, preventing undefined metrics and ensuring reliable performance estimates. Critical for imbalanced datasets like clinical trials.[8]
  • Primary Metric: ROC AUC (handles class imbalance) i Why ROC AUC?
    Recent research (2024) shows ROC AUC is robust to class imbalance, measuring discriminative ability regardless of classification threshold. Ideal for comparing model performance on imbalanced clinical trial data.[9]
  • Secondary Metrics: Precision, Recall, F1, Accuracy

Key Findings

Critical insights from model evaluation and comparison

Model Performance Insights

Best Model: Gradient Boosting achieved the highest ROC AUC of 0.8243, demonstrating superior discriminative ability in predicting trial outcomes.

Ensemble Methods Dominance: Random Forest, Gradient Boosting, and XGBoost consistently outperformed simpler models, highlighting the value of ensemble approaches for this complex prediction task. i Why Ensemble Methods Excel:
Random Forest reduces variance through parallel tree averaging, while gradient boosting (XGBoost, GB) reduces bias by sequentially correcting errors. XGBoost adds regularization for better generalization, explaining its typical performance advantage.[10]

Cross-Validation Consistency: All models showed stable performance across folds with low standard deviations, indicating robust and reliable predictions that generalize well.

Infrastructure Efficiency: Parallel training using GitHub Actions matrix strategy reduced total pipeline execution time from ~28 minutes (sequential) to ~12 minutes (parallel), enabling faster iteration cycles.

Real-World Impact: With pharmaceutical Phase 2/3 trials costing $10-100M+[2] and 70% Phase 2 failure rate[1], even modest improvements in prediction accuracy can guide resource allocation and save millions in failed trial investments[4].
